SVG Template Studio

analyze → schema → data → preview → batch export

1. Load your SVG template

Paste SVG markup or upload a .svg file. Tag any element you want to be data-driven with an id attribute — text, image, or shape. The Studio reads those ids to build your schema.

Click to upload an .svg file, or drop one here — or just paste markup below
Convention tip: use plain ids like name time effect art. Add data-type="text|number|color|image" on an element to override the auto-detected field type.

SVG text never wraps automatically — add data-wrap-width="480" on a <text> element to auto-wrap long values to that many px wide. Optional: data-line-height="26" (defaults to ~1.25× font size) and data-max-lines="3" (truncates with "…" past that many lines).

2. Generated schema

Every tagged element became a field. Adjust the type if the auto-detection guessed wrong — this decides what kind of editor you get in the data step. Exclude a field to hide it from the data editor — it'll always render using its default value instead.

3. Fill in your data

Each row becomes one generated card. Add rows by hand, or import a JSON array / CSV that matches your schema keys.

4. Preview

Check how each row renders before batch generating everything.
